Capítulo 06 — 2010 hasta hoy

La Web Moderna

La web fue construida para documentos. Luego se convirtió en una plataforma de aplicaciones. Luego en el sistema operativo. Hoy, la línea entre un "sitio web" y una "aplicación nativa" es principalmente una cuestión de distribución. Entender la web moderna significa entender cómo llegamos hasta aquí — y ser honestos sobre lo que quizás hemos sobrecomplicado por el camino.

HTML5: Una Plataforma, No Solo Nuevas Etiquetas

HTML5, publicado oficialmente en 2014 (con los navegadores implementándolo durante años antes), no era solo una nueva versión de HTML. Era una colección de APIs que convirtieron el navegador en un entorno de ejecución de aplicaciones real.

  • API Canvas — gráficos 2D y 3D programáticos
  • Web Storage — datos persistentes sin cookies
  • WebSockets — comunicación bidireccional en tiempo real
  • Geolocalización — ubicación del dispositivo con permiso del usuario
  • Audio/video nativo — la sentencia de muerte de Flash
  • Service Workers — scripts en segundo plano, soporte offline, la base de las PWAs
  • Web Workers — hilos en segundo plano en JavaScript
Curiosidad

El primer borrador de HTML5 fue publicado en 2008. La especificación no se finalizó hasta el 28 de octubre de 2014 — seis años después. El W3C lo celebró tuiteando que HTML5 estaba "listo." Los desarrolladores lo celebraron señalando que los navegadores ya lo llevaban usando años. Los organismos de estándares se mueven despacio. Los navegadores se mueven primero y estandarizan después. Siempre ha sido así.

El Móvil Reescribe las Reglas

El iPhone se lanzó en 2007. Android en 2008. Para 2013, el tráfico móvil había superado al de escritorio en muchos sitios importantes. La web había sido construida para monitores grandes y entrada por ratón. De repente tenía que funcionar en una pantalla táctil de 8 centímetros.

Ethan Marcotte acuñó el Diseño Web Responsivo en 2010: un único código que se adapta a cualquier tamaño de pantalla mediante media queries de CSS. En dos años era el enfoque predeterminado. El diseño mobile-first — diseñar para pantallas pequeñas primero — se convirtió en la metodología dominante para 2015. 

/* El media query que cambió cómo construimos todo */
@media (max-width: 768px) {
  .contenedor {
    flex-direction: column;
    padding: 16px;
  }
}

Las Guerras de Frameworks de JavaScript

A medida que las aplicaciones web crecían en complejidad, jQuery + HTML renderizado en servidor mostró sus límites. Gestionar estado, mantener la interfaz sincronizada con los datos, manejar el enrutamiento y la reutilización de componentes — eran problemas resolubles, pero jQuery no era la herramienta adecuada.

Una nueva clase de herramientas tomó el control completo del renderizado:

  • Backbone.js (2010) — primer framework MVC importante para el frontend
  • AngularJS (2010) — el framework de Google, enlace de datos bidireccional, opinionado
  • React (2013) — la biblioteca de Facebook, modelo de componentes, DOM virtual, flujo de datos unidireccional
  • Vue (2014) — más ligero, más accesible, adorado por desarrolladores independientes
  • Svelte (2016) — compila componentes en tiempo de construcción, no envía runtime
  • Next.js (2016) — React con renderizado en servidor, enrutamiento y despliegue integrados

El modelo de componentes de React ganó. La idea: la interfaz es una función pura del estado. Cambia el estado, la interfaz se actualiza automáticamente. Menos bugs, modelo mental más claro.

Curiosidad

React fue creado por Jordan Walke en Facebook, desplegado por primera vez en el News Feed de Facebook en 2011 y publicado como código abierto en 2013. Fue recibido inicialmente con escepticismo — mezclar HTML y JavaScript en JSX era visto como retrógrado y equivocado. En tres años era la biblioteca de interfaz dominante en la web. A veces lo que peor parece al principio gana porque está resolviendo el problema correcto.

Node.js: JavaScript Escapa del Navegador

En 2009, Ryan Dahl lanzó Node.js — un entorno de ejecución de JavaScript para el servidor. La propuesta: un solo lenguaje en todas partes. El mismo desarrollador que escribe tu frontend en React puede escribir tu backend. Sin cambio de contexto entre Python y JS.

El modelo de I/O no bloqueante y orientado a eventos de Node lo hacía especialmente bueno para manejar muchas conexiones concurrentes — aplicaciones en tiempo real, APIs, streaming. npm, el registro de paquetes de Node, se convirtió en el mayor registro de software del mundo.

HTTPS: La Web Finalmente se Cifra

En la web temprana, HTTPS era solo para bancos y páginas de pago. Todo lo demás era HTTP plano — sin cifrar, legible por cualquiera en el camino entre tú y el servidor. Tu ISP. El router de una cafetería. Un gobierno.

En 2014, Google dijo que HTTPS sería una señal de posicionamiento. En 2015, se lanzó Let's Encrypt — certificados SSL gratuitos y automatizados. En 2018, Chrome marcó todos los sitios HTTP como "No Seguro." Hoy HTTPS es el estándar. La web está cifrada por primera vez en su historia.

La Contrarevolución: Vuelta a la Simplicidad

Para 2020, la complejidad era innegable. Un proyecto web "simple" requería webpack, Babel, ESLint, TypeScript, un package.json con 200 dependencias, un pipeline de CI/CD y una plataforma de despliegue. La cadena de herramientas se había convertido en el proyecto.

Surgió un movimiento contrario: HTMX, Astro, 11ty — enfoques que cuestionaban si toda esa complejidad de JavaScript era realmente necesaria. ¿Podríamos construir mejor volviendo al modelo original de la web — HTML, HTTP, hipervínculos?

El Círculo se Cierra

HTMX y el Regreso a los Hipermedia

HTMX extiende HTML con atributos que permiten a cualquier elemento hacer peticiones HTTP e intercambiar contenido en la página. Sin JavaScript necesario para la mayoría de interacciones. El servidor envía HTML. El navegador renderiza HTML. Como siempre funcionó.

Es una filosofía: el modelo hipermedia original de la web era correcto. Inventamos un problema (la SPA de JavaScript) y luego construimos soluciones elaboradas para él. HTMX pregunta: ¿y si no lo hubiéramos hecho?

Este sitio está construido con HTMX. A propósito.

Fechas Clave
7 jun. 2010

Steve Jobs publica "Pensamientos sobre Flash." Flash no estará en el iPhone. Comienza la muerte de la era de los plugins.

9 may. 2013

React publicado como código abierto por Facebook. JSX es ridiculizado. En 3 años es la biblioteca de interfaz más popular de la web. El modelo de componentes gana.

28 oct. 2014

HTML5 finalizado. Los navegadores llevaban implementándolo 6 años. La especificación se pone al día con la realidad.

Abr. 2016

Let's Encrypt sale de beta. SSL gratuito para todos. La web cifrada se convierte en la web predeterminada. HTTPS pasa del 30% al 80%+ del tráfico en 4 años.

31 dic. 2020

Flash es oficialmente eliminado por Adobe. Más de 15 años de juegos, animaciones y cultura de internet se apagan. El proyecto Flashpoint archiva más de 70.000 juegos Flash.

30 nov. 2022

ChatGPT se lanza. 1 millón de usuarios en 5 días. Comienza la era de la IA. El próximo capítulo de la web empieza a escribirse solo.

"La web no está terminada. Nunca se suponía que lo estuviera. Tim Berners-Lee diseñó un sistema que pudiera evolucionar. El hecho de que sigamos debatiendo la manera correcta de construir para ella es la prueba de que funciona exactamente como fue concebida."
