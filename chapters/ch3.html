Chapter 03 — 1993 to 2001

★ The Browser Wars ★

Mosaic made the web visual. Netscape made it commercial. Internet Explorer made it a battlefield. In eight chaotic years, the web went from an academic curiosity to a global phenomenon — and nearly tore itself apart in the process.

Mosaic: The Web Gets Eyes

In January 1993, Marc Andreessen and Eric Bina released Mosaic at the National Center for Supercomputing Applications (NCSA). It was the first browser that could display images inline with text — on the same page, not in a separate window.

The difference was immediately obvious. Suddenly the web looked like something. Mosaic was also the first browser to work on Windows and Mac, not just Unix. Within a year, there were over a million downloads.

The web had found its killer feature: it was a visual medium, not just a text medium. And now anyone could use it.

Netscape and the Commercial Web

In 1994, Marc Andreessen co-founded Netscape Communications. Netscape Navigator launched that year and quickly dominated the browser market — at its peak, over 80% of web users ran it. Netscape's IPO in 1995 was one of the largest in history to that point. A two-year-old company with no profits was valued at nearly $3 billion.

The dot-com era had officially begun. Investors poured money into anything with a .com in its name. The web was gold.

JavaScript: Born in 10 Days

In 1995, Netscape hired a developer named Brendan Eich. His assignment: create a scripting language for the browser. Time given: 10 days.

What emerged was JavaScript — a language that could run directly in the browser and manipulate the page. For the first time, HTML pages could respond to user actions. Buttons could do things. Forms could validate before submitting. Pages could update without reloading.

The 10-day origin story shows in the language's quirks — and there are many. But JavaScript's placement in the browser made it the most widely deployed programming language in history. You couldn't escape it even if you wanted to.

Historical Context

Why the Name "JavaScript"?

The original name was Mocha, then LiveScript. It was renamed JavaScript as a marketing move to ride the popularity of Java — even though the two languages are completely unrelated. Same relationship as "car" and "carpet." This naming decision created confusion that persists to this day.

Microsoft Enters the War

In 1995, Microsoft released Internet Explorer 1.0 — bundled for free with Windows 95. They had licensed Mosaic's code, built their own browser on top of it, and made it impossible to avoid: if you had Windows, you had IE.

What followed was the Browser War. Both Netscape and Microsoft started adding their own proprietary HTML tags and features. Pages built for Netscape broke in IE and vice versa. Developers had to build two versions of everything. The phrase "Best viewed in Netscape Navigator 4.0" became a fixture of the era.

By 1998, IE had surpassed Netscape in market share. In 2000, Netscape effectively died as a commercial product. Microsoft had won — but at a cost to the open web.

CSS: Separating Style from Structure

In the early web, styling was done with HTML directly. You'd use the <font> tag to set colors and sizes, the <center> tag for alignment, <blink> to make text flash (yes, really). The structure of the document was tangled up with its visual presentation.

In 1996, the W3C published the CSS 1 specification. Cascading Style Sheets separated design from content: HTML described structure, CSS described appearance. This was the right architectural decision. Browsers took years to implement it properly, but the principle endured. 

/* CSS changed everything */
body {
  background-color: #c0c0c0;
  font-family: Times New Roman;
  color: navy;
}
h1 { font-size: 24pt; }
a  { color: blue; text-decoration: underline; }

Table Layouts and the Dark Arts

CSS existed but browsers barely implemented it. So web designers got creative. The solution they found was using HTML <table> elements — meant for tabular data — to build page layouts. Columns, grids, nested boxes: all done with tables.

It was a hack. A creative, widespread, necessary hack. Every major website in the late 90s was built this way. The code was a nightmare of nested tables, invisible spacer GIFs, and hardcoded pixel values. But it worked.

The late 90s web was an aesthetic crime scene. And we loved it. Tiled backgrounds. Hit counters. Comic Sans. Animated GIFs. "Under Construction" banners. Guestbooks. The chaos of a medium that had no rules yet.

What the Browser Wars Gave Us

The competition was messy and harmful to developers, but it drove rapid innovation. Almost every major web technology we still use today — JavaScript, CSS, cookies, DHTML, SSL/HTTPS, forms — was created or popularized during this era.

The war also produced a hard-learned lesson: proprietary extensions to open standards fragment the web. Standards matter. A web where each browser speaks a different language is a web that barely works.

That lesson would take another decade to fully sink in — and the cleanup of the mess would fall to a browser that didn't even exist yet.
