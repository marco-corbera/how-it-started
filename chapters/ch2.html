Chapter 02 — 1989 to 1995

The First Web

In 1989, a physicist at CERN wrote a proposal called "Information Management: A Proposal." His boss wrote "Vague but exciting" on the cover page and returned it. That physicist was Tim Berners-Lee, and what he was proposing would eventually become the most transformative communication medium in human history.

The Problem at CERN

CERN — the European Organization for Nuclear Research — employed thousands of scientists. They came from different universities, used different computers, and constantly changed jobs. Every time someone left, their knowledge walked out the door with them. Documents existed but were scattered across incompatible systems. Finding information required knowing exactly where it was.

Tim Berners-Lee's insight was this: the internet already existed. Computers were already networked. What was missing was a system that let documents link to each other across different machines — so that reading one document could lead you naturally to related documents, regardless of where they lived.

He called this idea hypertext — text that contains links to other text. The concept wasn't new (Ted Nelson had theorized it in the 1960s), but Berners-Lee was the first to implement it over a network.

The Three Inventions

To make the web work, Berners-Lee needed to solve three problems: how to format documents, how to locate them, and how to transfer them. He invented three things:

Invention 1

HTML — HyperText Markup Language

A simple language for structuring documents. You mark up text with tags that describe what it is: a heading, a paragraph, a link, an image.

The killer feature: the <a> tag. A hyperlink. A way to point from one document to another, on the same machine or anywhere on the network. Click it and your browser fetches that document. That simple mechanic is what made the web a web.

Invention 2

URL — Uniform Resource Locator

A standardized address format for any resource on the internet. http://info.cern.ch/hypertext/WWW/TheProject.html — the first URL.

A URL has three parts: the scheme (http://) tells the browser what protocol to use; the host (info.cern.ch) is the domain resolved via DNS to an IP address; the path is the location of the resource on that server.

Invention 3

HTTP — HyperText Transfer Protocol

The protocol that defines how browsers and servers talk to each other. It's a simple request-response model: the browser sends a request, the server sends a response.

The first version, HTTP/0.9, was one line long: GET /page.html. That's it. The server would respond with the HTML content of the file. No headers. No status codes. Just ask, receive, done.

The First Website

On August 6, 1991, Tim Berners-Lee published the first website on the internet. It ran on a NeXT computer at CERN with the IP address 128.141.35.151. He taped a note to the machine: "This machine is a server. DO NOT POWER IT DOWN!!"

The website described the World Wide Web project itself — what it was, how it worked, how to set up a web server. Meta from day one.

The first browser — also built by Berners-Lee — was called WorldWideWeb (later renamed Nexus). It ran only on NeXT computers. It could display text and hyperlinks but not images. You would have to wait for Mosaic in 1993 to see the web as a visual medium.

What HTML Actually Is

HTML is not a programming language. You can't write logic in it. It doesn't make decisions or repeat operations. It's a markup language — it describes structure and meaning. 

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, World</h1>
    <p>This is a paragraph.</p>
    <a href="http://info.cern.ch">The first website</a>
  </body>
</html>

The browser reads this markup and decides how to display it. In 1993, that meant whatever the browser thought looked right — there were no stylesheets yet. Every browser rendered pages differently. That chaos would persist for years.

HTTP: How the Web Actually Works

HTTP is what makes the browser and the server dance. Every time you load a page, this conversation happens: 

-- Browser asks: --
GET /index.html HTTP/1.1
Host: example.com

-- Server responds: --
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>

The status code (200, 404, 500 etc.) tells the browser whether the request succeeded. 200 OK means all good. 404 Not Found means the resource doesn't exist. 500 Internal Server Error means something went wrong on the server side. These codes are part of the HTTP specification and every developer eventually memorizes the important ones.

A Crucial Decision: Making It Free

In April 1993, CERN made a decision that changed everything: they released the World Wide Web technology into the public domain, royalty-free. No license fees. No restrictions. Anyone could build a web server or browser.

This single decision is why the web exploded. If CERN had kept it proprietary — as they easily could have — the web might have become a niche academic tool. Instead, within two years, anyone with a computer could participate.

"The web is more a social creation than a technical one. I designed it for a social effect — to help people work together — and not as a technical toy."
— Tim Berners-Lee

What Was Still Missing

By 1993, the web existed but was barely usable for normal people:

  • No images in browsers (text only)
  • No design — just plain text on grey backgrounds
  • No CSS — styling didn't exist yet
  • No JavaScript — pages were completely static
  • You needed to know the exact URL to visit a site — no search engines yet
  • You needed to configure your computer manually to get online

Then, in 1993, a student at the University of Illinois released a browser that changed all of this. His name was Marc Andreessen, and his browser was called Mosaic. The web was about to become visual — and a war was about to begin.
