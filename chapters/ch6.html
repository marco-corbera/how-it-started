Chapter 06 — 2010 to Now

The Modern Web

The web was built for documents. Then it became an application platform. Then it became the operating system. Today, the line between a "website" and a "native application" is mostly a question of distribution. Understanding the modern web means understanding how we got here — and being honest about what we may have overcomplicated along the way.

HTML5: A Platform, Not Just New Tags

HTML5, published officially in 2014 (with browsers implementing it for years prior), was not just a new version of HTML. It was a collection of APIs that turned the browser into a proper application runtime.

  • Canvas API — programmatic 2D and 3D graphics
  • Web Storage — persistent data without cookies
  • WebSockets — real-time bidirectional communication
  • Geolocation — device location with user permission
  • Native audio/video — Flash's death sentence
  • Service Workers — background scripts, offline support, the foundation of PWAs
  • Web Workers — background threads in JavaScript
Curious fact

The first draft of HTML5 was published in 2008. The spec wasn't finalized until October 28, 2014 — six years later. The W3C celebrated by tweeting that HTML5 is "done." Developers celebrated by pointing out that browsers had been using it for years already. Standards bodies move slowly. Browsers move first and standardize later. They always have.

Mobile Rewrites the Rules

The iPhone launched in 2007. Android in 2008. By 2013, mobile traffic had overtaken desktop on many major sites. The web had been built for big monitors and mouse input. Suddenly it had to work on a 3-inch touchscreen.

Ethan Marcotte coined Responsive Web Design in 2010: one codebase that adapts to any screen size via CSS media queries. Within two years it was the default approach. Mobile-first design — designing for small screens first — became the dominant methodology by 2015. 

/* The media query that changed how we build everything */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
    padding: 16px;
  }
}

The JavaScript Framework Wars

As web apps grew in complexity, jQuery + server-rendered HTML showed its limits. Managing state, keeping UI in sync with data, handling routing and component reuse — these were solvable problems, but jQuery wasn't the right tool for them.

A new class of tools took over rendering entirely:

  • Backbone.js (2010) — first major MVC framework for the frontend
  • AngularJS (2010) — Google's framework, two-way data binding, opinionated
  • React (2013) — Facebook's library, component model, virtual DOM, unidirectional data flow
  • Vue (2014) — lighter, more approachable, beloved by solo devs
  • Svelte (2016) — compiles components at build time, ships no runtime
  • Next.js (2016) — React with server rendering, routing, and deployment baked in

React's component model won. The idea: UI is a pure function of state. Change the state, the UI updates automatically. Fewer bugs, clearer mental model.

Curious fact

React was created by Jordan Walke at Facebook, first deployed on Facebook's News Feed in 2011, and open-sourced in 2013. It was initially met with skepticism — mixing HTML and JavaScript in JSX was seen as backwards and wrong. Within three years it was the dominant UI library on the web. Sometimes the thing that looks worst at first wins because it's solving the right problem.

Node.js: JavaScript Escapes the Browser

In 2009, Ryan Dahl released Node.js — a JavaScript runtime for the server. The pitch: one language everywhere. The same developer who writes your React frontend can write your backend. No context switch between Python and JS.

Node's event-driven, non-blocking I/O made it particularly good at handling many concurrent connections — real-time apps, APIs, streaming. npm, Node's package registry, became the largest software registry in the world.

HTTPS: The Web Finally Gets Encrypted

In the early web, HTTPS was only for banks and checkout pages. Everything else was plain HTTP — unencrypted, readable by anyone in the path between you and the server. Your ISP. A coffee shop router. A government.

In 2014, Google said HTTPS would be a ranking signal. In 2015, Let's Encrypt launched — free, automated SSL certificates. In 2018, Chrome marked all HTTP sites "Not Secure." Today HTTPS is the default. The web is encrypted for the first time in its history.

The Counter-Revolution: Back to Simplicity

By 2020, the complexity had become undeniable. A "simple" web project required webpack, Babel, ESLint, TypeScript, a package.json with 200 dependencies, a CI/CD pipeline, and a deployment platform. The toolchain had become the project.

A counter-movement emerged: HTMX, Astro, 11ty — approaches that questioned whether all this JavaScript complexity was actually necessary. Could we build better by returning to the web's original model — HTML, HTTP, hyperlinks?

Full Circle

HTMX and the Return to Hypermedia

HTMX extends HTML with attributes that let any element make HTTP requests and swap content into the page. No JavaScript required for most interactions. The server sends HTML. The browser renders HTML. The way it always worked.

It's a philosophy: the web's original hypermedia model was correct. We invented a problem (the JavaScript SPA) and then built elaborate solutions for it. HTMX asks: what if we hadn't?

This site is built with HTMX. On purpose.

Key Dates
Jun 7, 2010

Steve Jobs publishes "Thoughts on Flash." Flash will not be on iPhone. The death of the plugin era begins.

May 9, 2013

React open-sourced by Facebook. JSX is mocked. Within 3 years it's the most popular UI library on the web. The component model wins.

Oct 28, 2014

HTML5 finalized. Browsers had been implementing it for 6 years already. The spec catches up with reality.

Apr 2016

Let's Encrypt exits beta. Free SSL for everyone. The encrypted web becomes the default web. HTTPS goes from 30% to 80%+ of traffic within 4 years.

Dec 31, 2020

Flash is officially killed by Adobe. 15+ years of browser games, animations, and internet culture go dark. The Flashpoint project archives 70,000+ Flash games.

Nov 30, 2022

ChatGPT launches. 1 million users in 5 days. The AI era begins. The web's next chapter starts writing itself.

"The web is not finished. It was never supposed to be. Tim Berners-Lee designed a system that could evolve. The fact that we're still arguing about the right way to build for it is proof it's working exactly as intended."
